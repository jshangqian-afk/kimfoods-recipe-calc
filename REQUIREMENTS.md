# キムチ製造電卓アプリ 要件定義書

KimFoods Co., Ltd. 自社工場向け / 製造記録・原材料分析システム

版: v1.0（Claude Code 開発着手用ドラフト）

---

## 1. このアプリの目的

現場ユーザーが**白菜・大根・チャンジャのキロ数を入力するだけ**で、その製品に必要なタレの種類・量と全副材料が自動計算・表示される、KimFoods 専用の製造電卓。

同時に、入力した1バッチ分の製造内容（原材料・タレ種類・副産物）をバックエンドのスプレッドシートに記録として保存し、デイリー / ウィークリー / マンスリー / イヤーで集計・分析できるようにする。最終的に原材料の使用状況を期間分析し、**仕入れデータ**としても活用できることをゴールとする。

---

## 2. 技術構成

既存の KimFoods アプリ群（kimfoods-stock, sanada-schedule 等）と同じ構成に揃える。

- フロントエンド: HTML + JavaScript（GitHub Pages でホスト）
- バックエンド: Google Apps Script（GAS）
- データストア: **新規** Google スプレッドシート（このアプリ専用に1つ用意）
- 想定デバイス: iPad / タブレット中心。タッチ操作前提で大きめのボタン・数字入力UI
- リポジトリ: 新規（命名例 `kimfoods-recipe-calc`、確定は後続）

---

## 3. 利用フロー（現場ユーザー視点）

1. 製品ボタンを1つタップして選択する
2. その製品の入力基準（白菜 / 大根 / チャンジャ）のキロ数だけを入力する
3. 必要なタレの種類・量と全副材料が自動表示される
4. 内容を確認し「記録」ボタンで保存する
5. 保存後も、記録した内容は**編集できる**（数値の打ち間違い・修正に対応）

入力欄は**キロ数のみ**。それ以外の材料量は全て自動計算で表示する。

### 4.4 計算係数の定義（コード実装用）

白菜基準（中辛 / 乳酸菌 / 大辛 / 手しごと白菜 / ヤマダ白菜）:
- タレ = 白菜kg × 0.25
- 大根 = 白菜kg × 0.14
- 人参 = 白菜kg × 0.03
- ニラ = 白菜kg × 0.04
- 昆布 = 白菜kg ÷ 20 → 一握り単位（目安表示）

大根基準（カクテギ午前/午後 / 手しごと大根 / ヤマダ大根）:
- タレ = 大根kg × 0.1
- 昆布 = 大根kg ÷ 40 → 一握り単位（目安表示）

チャンジャ基準:
- Bタレ = チャンジャkg × 0.57
- 大辛パウダー = チャンジャkg × 3g/kg（= チャンジャkg × 3 g）
- ごま油 = 20kgあたり大さじ3（比例表示）
- ごま = 20kgあたり大さじ2（比例表示）

製品固有の追加材料:
- 大辛パウダー（大辛）= 白菜kg × 17g/kg（= 白菜kg × 17 g）
- 砂糖（ヤマダ白菜・ヤマダ大根）= タレ重量 × 0.19

> 注: 昆布「一握り」、ごま油「大さじ3」等の目安系は厳密重量化せず、目安表示のまま記録する（後述）。

## 6. 記録（バッチ）仕様

### 6.1 1レコードの単位

**1バッチ = 1レコード**。1日に同じ製品を複数回作る場合は、その都度1件ずつ記録する。

### 6.2 レコードの識別

二層構造で管理する。

- **内部ID（一意・重複なし）**: `YYYYMMDD-<製品コード>-<連番3桁>`
  例: `20260525-nakakara-001`
- **表示ラベル（現場が見て分かる）**: 日付 + 製品 + 時間帯 + 「その日・その製品の通し番号（◯回目）」

連番は「同じ日・同じ製品」の中で自動採番する。現場ユーザーは回数を意識せず、記録ボタンを押すたびにシステムが自動で 1回目→2回目→… と振る。

### 6.3 記録イメージ

| 日付 | 製品 | 時間帯 | バッチ | 内部ID |
|---|---|---|---|---|
| 2026/5/25 | 中辛 | 午前 | 1回目 | 20260525-nakakara-001 |
| 2026/5/25 | 中辛 | 午前 | 2回目 | 20260525-nakakara-002 |
| 2026/5/25 | 中辛 | 午後 | 1回目 | 20260525-nakakara-003 |
| 2026/5/25 | カクテギ | 午前 | 1回目 | 20260525-kakutegi-001 |

### 6.4 編集

保存済みレコードは内部IDで特定し、数値・内容を編集できる。編集時に他のレコードへ影響を与えない（内部IDで安全に1件のみ更新）。

記録一覧から2段階確認後に1件を削除できる。削除後もバッチ番号は再利用せず、同日・同製品の最大番号の次を採番する。

## 7. バックエンド（スプレッドシート）設計案

新規スプレッドシートに以下のシートを用意する。

### 7.1 records シート（生データ / 全バッチの明細）

1バッチ1行。列案:

| 列 | 内容 |
|---|---|
| record_id | 内部ID（一意） |
| date | 製造日（YYYY/MM/DD） |
| product | 製品名（中辛 等） |
| product_code | 製品コード（nakakara 等） |
| time_slot | 時間帯（午前 / 午後 / なし） |
| batch_no | その日その製品の通し番号 |
| base_material | 入力基準（白菜 / 大根 / チャンジャ） |
| base_kg | 入力キロ数 |
| tare_type | タレ種類（A / B / C） |
| tare_kg | タレ量 |
| daikon_kg | 大根量 |
| ninjin_kg | 人参量 |
| nira_kg | ニラ量 |
| konbu | 昆布（目安） |
| daikara_powder_g | 大辛パウダー（g） |
| sugar_kg | 砂糖量 |
| sesame_oil | ごま油（目安） |
| sesame | ごま（目安） |
| created_at | 記録日時 |
| updated_at | 更新日時 |

> 製品によって使わない列は空欄。GAS 側で製品ごとに必要な値だけ埋める。

### 7.2 集計シート（自動更新）

records を元に GAS / 数式で自動集計。

- daily（日次）: 日 × 製品 × 原材料使用量
- weekly（週次）
- monthly（月次）
- yearly（年次）

集計軸: 製品別 / 原材料別 / タレ種類別 / 時間帯別。

### 7.3 分析・仕入れ用アウトプット

月間・週間・年間で原材料（白菜・大根・人参・ニラ・タレ・パウダー・砂糖・ごま油 等）の総使用量を出し、**仕入れ計画データ**として参照・書き出しできる形にする。

## 8. UI 方針

- iPad / タブレット最適化。大きめボタン・大きめ数字入力（カスタムテンキー検討）
- タッチ操作は44px以上を基本とし、iPad Safariの動的画面高とセーフエリアを考慮する
- 画面遷移: ①製品選択 → ②キロ数入力 → ③計算結果確認 → ④記録 → ⑤一覧 / 編集
- 出力された UI の見た目そのままに記録内容が並ぶ（UI = 記録内容が一致）
- **本番着手前に UI プロトタイプを複数案つくり、間違いがないか確認してから実装する**

## 11. v1.1 追加仕様（2026-06-22）

- 今日の白菜予定は、大樽210kg・小樽90kg・100樽35kgの個数と「昨日の残り」手入力値を合算する。
- 樽は各種類を任意個数組み合わせられ、予定は日付ごとにスプレッドシートへ保存する。
- 本日の残りは「予定合計 − 当日の保存済み白菜基準レコードのbase_kg合計」。レコード編集後も再計算する。
- 製品ごとに商品内容量（g）を管理画面で登録・編集する。
- 予定数は整数切り捨て。白菜は原料kg×1.46、大根は×1.1、チャンジャは×1.57を商品内容量kgで割る。
