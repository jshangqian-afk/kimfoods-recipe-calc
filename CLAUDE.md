# CLAUDE.md — キムチ製造電卓アプリ 開発規約

このファイルは Claude Code がリポジトリを開いたとき最初に読む、プロジェクトの「どう作るか」を定義するものです。
「何を作るか」は `REQUIREMENTS.md` を、配合係数の正本は `recipes.js` を参照してください。

---

## プロジェクト概要

KimFoods 自社工場向けのキムチ製造電卓 + 製造記録システム。
現場ユーザーが製品ボタンを選び、白菜 / 大根 / チャンジャのキロ数を入力すると、
必要なタレ種類・量と全副材料が自動計算・表示され、1バッチ単位で記録を保存する。
記録はスプレッドシートに蓄積し、日次 / 週次 / 月次 / 年次で集計・分析し、仕入れデータ化する。

---

## 技術スタック（固定）

- フロントエンド: 素の HTML + CSS + JavaScript（フレームワーク不使用）
- ホスティング: GitHub Pages
- バックエンド: Google Apps Script（GAS）/ clasp で管理
- データストア: 新規 Google スプレッドシート（このアプリ専用）
- 対象デバイス: iPad / タブレット中心（タッチ前提）

開発パイプライン: Claude Code + GitHub CLI + clasp（既存 KimFoods アプリと同様）。

---

## 最重要ルール

1. **配合係数は `recipes.js` を唯一の正本とする。**
   係数（×0.25 等）をフロントや GAS のコードに直書きしない。必ず `recipes.js` を import / 参照する。
   係数変更時は `recipes.js` だけを編集すれば全体に反映される構造を保つ。

2. **表示と記録は同じ計算結果を使う。**
   画面に出した値と、スプレッドシートに保存する値が必ず一致すること。
   `calcRecipe()` の戻り値を表示にも記録にも使い、二重計算しない。

3. **スプレッドシートの列マッピングは定義書に従う。**
   `SPREADSHEET.md` の列順・列名を正とする。列の追加・並べ替えをコード側で勝手に行わない。
   （過去に行マッピングずれのバグ事例あり。ヘッダー名でのマッピングを推奨。）

4. **記録の一意性は内部IDで担保する。**
   `record_id = YYYYMMDD-<product_code>-<連番3桁>`。
   編集・削除は必ず record_id で1件特定して行い、他レコードに影響を与えない。

---

## コーディング規約

- UI 文言・コメントは日本語。
- 数値は表示前に必ず丸める（`round()` 経由）。浮動小数の誤差をそのまま画面に出さない。
- iPad 前提のため、ボタン・入力欄は大きめ。数値入力はカスタムテンキーを検討。
- 変更時は**全文を出力**する（差分だけでなくファイル全体）。
- localStorage / sessionStorage には依存しない（記録は必ず GAS 経由でスプレッドシートへ）。
- GAS の Web API は doGet / doPost で受け、JSON で返す。

---

## 画面構成

1. 製品選択（ボタン: メイン9 + ヤマダ別枠2）
2. キロ数入力（基準材料の kg のみ）
3. 計算結果確認（タレ種類・量 + 全副材料）
4. 記録保存
5. 記録一覧 / 編集

メイン製品ボタンの並び順は `recipes.js` の `order` に従う。
